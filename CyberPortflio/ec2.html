AWS EC2 Instance Hardening

Objective: Deploy and secure an AWS Free Tier EC2 instance using blue-team practices.

Executive Summary

This project demonstrated the deployment and hardening of an AWS EC2 instance using Free Tier resources. The focus was on applying essential cloud security practices that directly map to production environments, including restricted network access, the principle of least privilege, and audit logging.

Scope & Objectives

  • Deploy an EC2 instance within AWS Free Tier.
  • Apply AWS security best practices to reduce risks of unauthorised access, credential exposure, and lack of monitoring.
  • Validate security measures through configuration checks and functional tests.

Key Risks Identified

  • Overly permissive inbound rules.
  • Credential exposure without IAM roles.
  • Lack of logging and monitoring.

Mitigation Measures Implemented

  • Restricted inbound access to SSH and optional HTTP/HTTPS.
  • Attached least privilege IAM role (CloudWatchAgentServerPolicy).
  • Enabled CloudTrail logging with logs stored in a secured S3 bucket.

Validation & Findings

Security group blocked all unauthorised ports, logging worked without embedded credentials, and CloudTrail provided a full audit trail.

Tools Used

AWS Management Console, Amazon EC2, IAM, CloudTrail, S3

Skills Demonstrated

Cloud security fundamentals, security group design, IAM role configuration, audit logging, risk assessment and remediation.

Conclusion

The EC2 instance was hardened against common threats, showing the application of AWS best practices in a real-world scenario.
