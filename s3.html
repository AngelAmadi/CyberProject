Securing Amazon S3 Bucket

Technical Summary

This project demonstrated the application of core cloud security practices by securing an Amazon S3 bucket against unauthorised access, data breaches, and accidental data loss. Key controls included enabling Block Public Access, enforcing default encryption, and turning on versioning. The outcome was a secure and resilient storage configuration that aligns with AWS best practices and real-world security requirements.

Scope & Objectives

  • Secure an S3 bucket using AWS-native controls
  • Prevent public exposure of sensitive data
  • Protect against accidental deletion or modification of stored objects
  • Demonstrate data confidentiality through encryption

Key Risks Identified

  • Public Exposure of Data: Misconfigured buckets could allow unauthorised internet access
  • Data Loss or Corruption: Accidental deletion or overwrites without recovery options
  • Unencrypted Data at Rest: Lack of encryption increases compromise risk

Mitigation Measures

  • Block Public Access: Enabled across the bucket to eliminate risk of unauthorised public access
  • Versioning: Activated to provide recovery options for deleted or modified objects
  • Default Encryption (SSE-S3): Applied to ensure all objects stored are encrypted automatically

Validation & Findings

  • Direct public access attempt confirmed blocked
  • Deleted object successfully restored via versioning
  • Objects verified as encrypted with SSE-S3

Recommendations

  • Perform regular configuration reviews
  • Apply least privilege access using IAM roles and policies
  • Enable CloudTrail logging to monitor access attempts
  • Consider automation with CloudFormation or Terraform
  • Enable MFA for privileged actions such as bucket deletion

Skills Demonstrated

AWS S3 security configuration, access control, encryption, resilience design, threat identification, and professional reporting

Conclusion
